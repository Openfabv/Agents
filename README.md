Openfab: Open Source Chip Design, Powered by AI

Welcome to Openfab, a pioneering platform where Open Source hardware meets advanced artificial intelligence.
Our mission is to democratize chip design by combining the collaborative power of Open Source with the intelligence and acceleration of Large Language Models (LLMs). Openfab is building the foundational AI and tools necessary to make custom silicon accessible, understandable, and rapid to develop for everyone.


What is Openfab?

Openfab (Open + Fabrication) is the interface between the existing Open Source chip design ecosystem (RISC-V, OpenROAD, etc.) and modern LLMs. We leverage a custom LLM fine-tuned on code, datasheets, RTL, verification protocols, and manufacturing process information.





Our platform aims to:

   Generate and refine RTL: Use natural language to describe digital circuits and let our AI assist with code generation, optimization, and synthesis.

   Accelerate physical design: Optimize standard cell placement and routing (P&R) to improve Power, Performance, and Area (PPA) results.

   Enhance verification: Automatically generate testbenches, identify edge cases, and assist with debugging circuit designs.

   Navigate the foundry process: Help interpret and apply complex design-rule-check (DRC) manuals for Open Source manufacturing processes.

   



Project Status

This is an early-stage project. We are actively developing:
    Openfab-LLM-v0.1: Training a specialized LLM fine-tuned on chip design data.
    Synthesis-Copilot: An API and CLI for interacting with our AI to generate Verilog from specifications.
    Community Hub: Building a library of pre-verified, AI-assisted Open Source IP blocks.





Roadmap

   Q4 2023: Initial model release, focusing on RTL generation.

   Q1 2024: Closed Beta of the Openfab synthesis platform.

   Q2 2024: Launch of Open Source verification and physical design modules.

   Q3 2024: Integration with existing Open Source tools (e.g., KLayout, Verilator).

   Q4 2024: Full public platform release.

  Model Training: Experts in fine-tuning large language models for domain-specific tasks.

  Hardware Design: Engineers to design and verify new Open Source IP blocks.

  UI/UX: Building a clean interface for an entirely new kind of chip design experience.
