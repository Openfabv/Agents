# Contributing to Openfabv Agents

Thank you for your interest in contributing to the Openfabv chip design agent! We welcome contributions from researchers, engineers, and enthusiasts in hardware design and AI.

## Getting Started

1. **Fork the repository** and clone it locally
2. **Create a feature branch** (`git checkout -b feature/your-feature`)
3. **Make your changes** following our code standards
4. **Test thoroughly** before submitting
5. **Submit a Pull Request** with clear description

## Areas We Need Help With

- **HDL Generation**: Improving Verilog/VHDL code generation
- **Error Correction**: Enhancing error detection and fix mechanisms
- **Optimization**: Design parameter optimization algorithms
- **Testing**: Writing test cases and validation scripts
- **Documentation**: Guides, tutorials, and API documentation
- **Examples**: Real-world chip design examples

## Code Standards

- Write clean, well-documented code
- Follow PEP 8 for Python, follow language conventions for others
- Include docstrings for functions and classes
- Add unit tests for new features
- Update documentation as needed

## Pull Request Process

1. Ensure your code passes all tests: `pytest tests/`
2. Update README if needed
3. Add description of changes and motivation
4. Link any related issues
5. Wait for review and address feedback

## Issue Guidelines

- Search existing issues before creating new ones
- Use clear, descriptive titles
- Provide minimal reproducible examples
- Include relevant system information

## Code of Conduct

Please be respectful and professional. We aim to create an inclusive community for all contributors.

## Questions?

- Open an issue for bugs or feature requests
- Start a discussion for questions about usage
- Check existing documentation first

Thank you for contributing! 🙏
